## 界面导览

### 1. 介绍页

介绍页是应用的首屏，展示完整的分析流程概览、AI 大模型配置入口以及全部工具的分类索引。

![介绍页](screenshots/局部截取_20260624_083521.png)

**功能说明：**

- **分析流程**：四步流程可视化（上传 → 解析 → 分析 → 报告），帮助用户理解整体工作流
- **AI 洞察设置**：左侧边栏配置 API Base URL、API Key 和模型名称（兼容 OpenAI 格式），供 AI 综合报告模块调用
- **工具索引**：按 8 大类别展示全部 14 个工具，点击可直接跳转至对应工具视图
- **文件上传区**：左上角上传 AF3 标准输出 `.zip` 压缩包（≤200MB）

**操作流程：**

1. 在左侧边栏点击 **Upload** 上传 AF3 输出 `.zip` 文件
2. （可选）在左下角 **AI 洞察设置** 中配置大模型 API 地址和密钥
3. 文件解析完成后，点击左侧 **仪表盘** 按钮进入分析界面

### 2. 仪表盘

仪表盘是分析工作的核心枢纽，以 KPI 卡片、3D 结构预览和工具卡片矩阵的形式全景展示分析状态。

![仪表盘 - KPI卡片与3D结构](screenshots/局部截取_20260624_124627.png)

**功能说明：**

- **KPI 卡片**（顶部四栏）：
  - **结构构象总数**：从 AF3 输出 JSON 中读取的构象数量
  - **最高置信度**：所有构象中的最高 ranking\_score
  - **项目名称**：可编辑的项目命名（默认读取文件名）
  - **已分析工具**：已完成分析的工具计数（如 0/14），切换工具后自动累加
- **3D 结构预览**：基于 py3Dmol 的蛋白质结构可视化，支持旋转、缩放
- **快捷按钮**：「一键报告」快速生成 AI 综合报告、「模型排行榜」查看各构象置信度排名
- **工具卡片矩阵**：按类别分组展示所有工具，已分析的工具标记为绿色，未使用的标记为灰色

![仪表盘 - 模型排行与工具卡片(上)](screenshots/局部截取_20260624_125400.png)

![仪表盘 - 工具卡片(中): 蛋白性质评估、成药口袋探测、结构相似性检索](screenshots/局部截取_20260624_125424.png)

![仪表盘 - 工具卡片(下): 蛋白功能注释、互作与3D可视化、AI综合报告](screenshots/局部截取_20260624_125444.png)

**操作流程：**

1. 上传 `.zip` 文件后自动进入仪表盘
2. 查看顶部 KPI 卡片确认数据解析成功
3. 浏览 3D 结构预览初步了解蛋白形态
4. 点击任意工具卡片或左侧快捷工具下拉菜单进入对应分析模块
5. 已分析的工具会显示绿色状态徽章，方便追踪进度

![全局基础分析 - pLDDT分布结果](screenshots/局部截取_20260624_125956.png)

### 4. 基础质量检查 (pLDDT)

解析 mmCIF 结构文件，提取 pLDDT 置信度分数并进行四级分级统计。

**功能说明：**

- 动态检测 mmCIF 列名（兼容 `label_atom_id` 和 `atom_name` 两种格式）
- pLDDT 四级分级：极高可信（>90）、高可信（70-90）、低可信（50-70）、极低可信（<50）
- 交互式 Plotly 饼图展示分布比例
- 结果自动缓存，供 AI 综合报告调用

**操作流程：**

1. 通过仪表盘工具卡片或左侧快捷工具进入 **基础质量检查**
2. 点击 **打开** 按钮
3. 系统自动解析 mmCIF 文件并展示统计结果

**预期结果：** 四级分级的残基数统计 + 交互式饼图（见全局基础分析结果截图）

**注意事项：**

- 本模块通常通过全局基础分析一键触发，也可单独运行
- AF3 mmCIF 格式使用 `label_atom_id` 列名，代码已自适应检测

### 5. PAE 误差矩阵分析

可视化 AlphaFold 3 预测的空间位置误差（Predicted Aligned Error）热力图。

![PAE误差矩阵分析](screenshots/局部截取_20260624_130109.png)

**功能说明：**

- 智能定位 PAE JSON 数据（自动扫描 ZIP 内嵌套目录）
- 生成 PAE 热力图：横轴为 Scored Residue，纵轴为 Aligned Residue，颜色映射误差值（Å）
- 统计指标：均值 PAE、中位数 PAE、最大值 PAE
- 低 PAE 值表示残基间相对位置预测可靠

**操作流程：**

1. 通过仪表盘或快捷工具进入 **PAE 误差矩阵分析**
2. 点击 **打开** 按钮
3. 系统自动解析 PAE 矩阵并渲染热力图

**预期结果：**

- PAE 热力图（蓝色 = 低误差，白色 = 高误差）
- 均值 / 中位数 / 最大值 PAE 统计（如均值 9.30Å、中位数 4.80Å、最大值 31.70Å）

**注意事项：**

- PAE 矩阵尺寸等于序列长度 × 序列长度（如 805×805）
- 对角线附近的低误差区域表示局部结构可靠
- 远离对角线的低误差区域表示长程相互作用预测可靠

### 6. 湿实验性质评估

评估靶点蛋白在体外表达纯化实验中的基本理化性质。

**功能说明：**

- **分子量（MW）**：基于氨基酸序列计算，单位 Da
- **等电点（pI）**：蛋白净电荷为零时的 pH 值
- **不稳定指数**：>40 判定为不稳定（红色预警标记），提示体外表达可能困难
- 结果自动缓存

**操作流程：**

1. 通过全局基础分析自动触发，或单独进入 **湿实验性质评估**
2. 系统自动从序列计算三项指标

**预期结果：**

- 分子量（如 92461.95 Da）
- 等电点（如 5.36）
- 不稳定指数（如 40.09，>40 显示红色"不稳定"标签）

**注意事项：**

- 不稳定指数 >40 仅提示体外表达可能存在困难，不代表蛋白无功能
- 计算基于氨基酸组成经验公式（Guruprasad 等方法）

### 7. 成药口袋探测（网格法）

基于网格法的蛋白表面成药结合口袋快速探测与 3D 可视化。

**功能说明：**

- 网格法扫描蛋白表面，识别潜在的结合口袋
- 计算口袋体积（Å³）、疏水性 GRAVY、网格点数
- 3D 球体可视化展示口袋空间位置（降采样至 2000 点防止浏览器卡顿）
- 纯 Python 实现，无需额外安装 C 引擎

**操作流程：**

1. 通过全局基础分析自动触发，或单独进入 **成药口袋探测**
2. 系统自动执行网格扫描并渲染 3D 可视化

**预期结果：**

- 口袋体积、疏水性 GRAVY、网格点数
- 3D 结构上的彩色球体标记口袋位置

**注意事项：**

- 网格法为快速近似方案，精度低于 fpocket C 引擎
- 如需更高精度，建议使用 fpocket 或 PrankWeb 模块

### 8. fpocket 高精度口袋探测

调用 fpocket C 引擎进行高精度几何口袋扫描。

![fpocket成药口袋探测](screenshots/局部截取_20260624_130206.png)

**功能说明：**

- 调用 fpocket C 引擎（需预先安装）进行 Voronoi 几何口袋扫描
- 自动将 mmCIF 转换为 PDB 格式供 fpocket 使用
- 输出详细口袋参数表：Pocket Score、Druggability Score、Alpha Spheres、SASA、Volume、Hydrophobicity
- 口袋 3D 可视化：在蛋白结构上以彩色球体标记各口袋位置

**操作流程：**

1. 确保已安装 fpocket（`conda install -c bioconda fpocket`）
2. 通过仪表盘或快捷工具进入 **fpocket 高精度口袋**
3. 点击 **打开** 按钮
4. 等待 fpocket 扫描完成（通常数秒至数十秒）

**预期结果：**

- 口袋参数表格（按 Druggability Score 排序）
- 口袋 3D 可视化图

**注意事项：**

- Windows 用户如未安装 fpocket，模块将优雅降级并提示使用网格法或 PrankWeb API
- Druggability Score >0.5 的口袋具有较高成药潜力

### 9. PrankWeb 口袋预测

通过 PrankWeb REST API 全自动进行机器学习口袋预测。

![PrankWeb口袋预测](screenshots/局部截取_20260624_130754.png)

**功能说明：**

- **全自动模式**：调用 PrankWeb API（`https://prankweb.cz/api/v2/`）自动上传结构并获取预测结果
- **算法**：P2Rank 机器学习模型（SOTA 性能）
- **输出**：口袋列表（得分、中心坐标、残基组成）
- **手动回退**：API 不可用时，提供 CASTp 手动桥接方案（下载转换后的 PDB → 上传 CASTp 网站 → 导入结果）

**操作流程：**

**全自动模式：**

1. 进入 **PrankWeb 口袋预测** 模块
2. 点击 **自动预测口袋 (PrankWeb)** 按钮
3. 等待 API 返回结果（通常 10-30 秒）

**手动回退模式（API 不可用时）：**

1. 展开 **CASTp 手动桥接** 面板
2. 点击 **下载 PDB** 获取转换后的结构文件
3. 访问 CASTp 网站上传分析
4. 将结果导入系统

**注意事项：**

- PrankWeb API 为外部服务，可能因服务器负载出现延迟或不可用
- 手动回退方案保留了 CASTp 作为备选

### 10. Foldseek 结构相似性检索

将本地结构发送至 EBI Foldseek 服务器，在 PDB 全库中检索结构同源蛋白。

![Foldseek结构检索 - 工具入口](screenshots/局部截取_20260624_130907.png)

**功能说明：**

- 自动将 mmCIF 转换为 PDB 格式
- 通过 Foldseek API 提交结构搜索任务
- 异步轮询获取搜索结果
- 输出 TM-score 排序的匹配列表（PDB ID、TM-score、序列一致性、E-value、比对长度）

**操作流程：**

1. 进入 **Foldseek 结构检索** 模块
2. 点击 **开始 Foldseek 全库搜索** 按钮
3. 等待搜索完成（通常 30-120 秒，取决于服务器负载）

**预期结果：**

![Foldseek结构检索 - 结果表格](screenshots/局部截取_20260624_131021.png)

- 匹配结构表格：PDB ID（可点击跳转）、TM-score、Seq Identity、E-value、Aln Len
- TM-score 接近 1.0 表示结构高度相似

**注意事项：**

- Foldseek 为外部 API 服务，搜索时间受服务器负载影响
- TM-score >0.5 通常认为结构显著相似
- 搜索结果中的 PDB ID 可点击跳转至 RCSB PDB 查看原始结构

### 11. Pfam 蛋白结构域注释

通过 EBI HMMER API 进行 Pfam 结构域快速注释。

![Pfam结构域注释 - 分析中](screenshots/局部截取_20260624_133843.png)

**功能说明：**

- 通过 EBI HMMER REST API（`https://www.ebi.ac.uk/Tools/hmmer/api/v1/`）进行异步结构域搜索
- 自动适配 2026 年 API 协议（`result` vs `results` 键名兼容）
- 异步提交 → 轮询 → 解析，最长等待 180 秒
- 输出结构域列表：Domain 名称、Description、Accession、Start/End 位置、E-value
- 结构域分布可视化图

**操作流程：**

1. 进入 **Pfam 结构域注释** 模块
2. 点击 **开始 Pfam 结构域注释** 按钮
3. 等待分析完成（长序列可能需要 1-3 分钟）
4. 查看结构域结果表格和分布图

**预期结果：**

![Pfam结构域注释 - 结果表格与分布图](screenshots/局部截取_20260624_141131.png)

- 结构域表格：Domain、Description、Accession、Start、End、E-value
- 结构域分布图：彩色条形图展示各结构域在序列上的位置

**注意事项：**

- 长序列（>500 aa）的 hmmscan 可能需要较长时间，请耐心等待
- 轮询超时（180秒）时建议稍后重试
- E-value 越小表示匹配越显著（通常 <1e-5 认为显著）

## 技术架构

### 数据流

```
用户上传 AF3 .zip 压缩包
        │
        ▼
  core/zip_parser.py
  （UUID 沙箱解压 → 深度扫描 → 数据提取）
        │
        ▼ session_state 持久化
┌───────────────────────────────────────────┐
│            三视图导航                       │
├────────────┬──────────────┬───────────────┤
│   介绍页    │    仪表盘     │    工具视图    │
│  (intro)   │ (dashboard)  │   (tool)     │
└────────────┴─────────────────────────────┘
        │            │              │
        │     ┌────────────┐       │
        │     │ 3D + 排行榜  │       │
        │     │ 分类工具卡片  │       │
        │     │ 状态徽章     │       │
        │     └──────┬──────┘       │
        └────────────┴──────────────┘
                         │
             ┌───────────┴───────────
             │    CacheManager        │
             │  数据 → 参数 → 元信息    │
             └───────────┬───────────┘
                         │
             ┌───────────┴───────────┐
             │  AI 综合报告（聚合）     │
             │  全量缓存 + 流式生成     │
             └───────────────────────┘
```

### 关键设计决策

| 决策               | 说明                                                     |
| :--------------- | :----------------------------------------------------- |
| **CacheManager** | 解决工具切换后分析结果丢失问题，支持 AI 报告聚合全量数据                         |
| **全局基础分析**       | 减少重复操作，一键完成最常用的三项分析                                    |
| **参数持久化**        | 用户调整的滑动条参数在切换工具后保持不变                                   |
| **CIF→PDB 自动转换** | fpocket 和 Foldseek 需要 PDB 格式，插件内置转换无需手动处理              |
| **HMMER 异步适配**   | 跟随 EBI 2026 API 协议更新，保证 Pfam 注释可用                      |
| **pLDDT 列名动态检测** | AF3 mmCIF 使用 `label_atom_id` 不同于旧版 `atom_name`，代码自适应检测 |
| **内存管理**         | 所有大数据对象使用后 `del` + `gc.collect()`，UUID 沙箱物理隔离          |
| **口袋 3D 可视化**    | 网格点降采样至 2000 点防止浏览器卡顿，半透明渲染展示口袋大致形态                    |

## 常见问题

| 问题                   | 状态         | 解决方案                                           |
| :------------------- | :--------- | :--------------------------------------------- |
| fpocket 未安装（Windows） | 已知限制       | 使用内置网格法「成药口袋探测」或 PrankWeb API                  |
| Pfam HMMER API 格式变更  | ✅ v3.1 已修复 | 已适配 EBI 2026 v1 异步协议                           |
| AI 流式报告内容丢失          | ✅ v3.0 已修复 | `st.empty()` 占位器逐块累积，替代 `st.write_stream()`    |
| pLDDT 列名不兼容          | ✅ v2.1 已修复 | `parser.py` 动态检测 `label_atom_id` / `atom_name` |
| 3D 视图未居中             | ✅ v3.0 已修复 | 添加 `zoomTo()` + `center()`                     |
| 口袋探测结果不渲染            | ✅ v3.0 已修复 | `pocket_result` 写入 `session_state`，3D 视图联动渲染   |
| ESMFold API 超时       | 速率限制       | 缩短序列长度或重试；回退到手动上传                              |
